# 阶段 6：嵌入式开发环境

## 目标

搭建嵌入式交叉编译与调试环境，掌握 ARM GCC 工具链、OpenOCD 调试器、Makefile 构建，以及 STM32 裸机开发入门。

## 前置依赖

- 阶段 3（C 语言、CMake 与 OpenCV）
- 阶段 1（Git）

## 任务

| 子项 | 任务 | 验收 |
|------|------|------|
| 6.1 ARM GCC 交叉编译链 | 在 MSYS2 UCRT64 终端中安装 `mingw-w64-ucrt-x86_64-arm-none-eabi-gcc`；编写一个简单的 C 程序，用 `arm-none-eabi-gcc` 交叉编译为 ARM 目标文件 | `arm-none-eabi-gcc --version` 输出正确；`file a.out` 显示为 ARM 架构 |
| 6.2 Makefile 构建 | 学习编写 Makefile（目标、依赖、变量、伪目标）；用 Makefile 管理交叉编译流程 | `make` 一次完成编译；`make clean` 清理产物 |
| 6.3 OpenOCD 调试 | 安装 [OpenOCD](https://openocd.org/)；连接开发板（如 STM32F4 Discovery）；读取设备 ID | `openocd -f <配置文件>` 成功识别目标芯片 |
| 6.4 STM32 裸机开发入门 | 使用 STM32CubeMX 生成初始化代码（或手动编写）；编写 LED 闪烁程序；编译并烧录到开发板 | 开发板上 LED 按预期闪烁 |

## 快速参考

```bash
# 安装 ARM GCC 交叉编译链（MSYS2 UCRT64）
pacman -S mingw-w64-ucrt-x86_64-arm-none-eabi-gcc

# 交叉编译示例
arm-none-eabi-gcc -mcpu=cortex-m4 -mthumb -c main.c -o main.o

# 基础 Makefile 模板
# TARGET = firmware
# CC = arm-none-eabi-gcc
# CFLAGS = -mcpu=cortex-m4 -mthumb -Wall -O2
# $(TARGET).elf: main.o
#     $(CC) $(CFLAGS) -o $@ $^
# clean:
#     rm -f *.o *.elf

# OpenOCD 调试
openocd -f interface/stlink.cfg -f target/stm32f4x.cfg
```
