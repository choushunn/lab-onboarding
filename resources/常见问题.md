# 常见问题与踩坑记录

## 阶段 0：网络与账号

### Clash Verge 安装后无法启动
- 尝试以管理员身份运行
- 检查系统防护软件是否拦截

### GitHub 无法访问
- 确认 Clash 已开启系统代理
- 在 Clash Verge 中开启「TUN 模式」

## 阶段 1：Git

### `git push` 提示 Permission denied
- 未配置 SSH 密钥，请执行 `ssh-keygen` 并添加到 GitHub
- 或切换为 HTTPS 方式：`git remote set-url origin https://...`

### `git commit` 提示 `please tell me who you are`
- 未配置 `user.name` 和 `user.email`，执行：
  ```
  git config --global user.name "你的名字"
  git config --global user.email "你的邮箱"
  ```

## 阶段 2：Python 与开发环境

### `conda` 命令找不到
- 确认已安装 Miniconda
- 重新打开终端，或手动执行 conda 的 `condabin/conda.bat`

### pip 安装速度慢
- 确认已配置清华源：`pip config set global.index-url https://pypi.tuna.tsinghua.edu.cn/simple`

### VSCode 找不到 Conda 解释器
- 按 `Ctrl+Shift+P` → `Python: Select Interpreter` → 选择对应的 Conda 环境

## 阶段 3：C 语言、CMake 与 OpenCV

### MSYS2 终端选错
- 安装 UCRT64 套件后，务必从开始菜单打开 **UCRT64** 终端，不是 MSYS2 或 MINGW32

### `gcc` 安装后提示找不到
- 确认已执行：`pacman -S mingw-w64-ucrt-x86_64-gcc`
- 确认当前使用的是 UCRT64 终端

### OpenCV cmake 配置失败
- 确认已安装所有依赖：`pacman -S mingw-w64-ucrt-x86_64-cmake mingw-w64-ucrt-x86_64-make`
- cmake 命令中必须加 `-G "MinGW Makefiles"`

## 阶段 4：Linux 虚拟机与容器

### 虚拟机无法联网
- VirtualBox 网络模式设为 **桥接网络** 或 **NAT**
- NAT 模式下宿主机无法直接 SSH 到 VM，需设置端口转发

### SSH 连接被拒绝
- VM 内执行 `sudo apt install openssh-server`
- 确认 SSH 服务已启动：`sudo systemctl status ssh`

### Docker 需要 sudo
- 将当前用户加入 docker 组：`sudo usermod -aG docker $USER`
- 重新登录生效

### Claude Code 安装失败
- 确认 Node.js 版本 >= 18：`node --version`
- 使用 `npm install -g @anthropic-ai/claude-code` 全局安装

## 阶段 5：AI / 深度学习环境

### `nvidia-smi` 命令找不到
- 确认已安装 NVIDIA 显卡驱动
- 检查驱动是否与当前 GPU 型号兼容

### PyTorch 安装后 CUDA 不可用
- 确认 CUDA 版本与 PyTorch 版本匹配：`python -c "import torch; print(torch.version.cuda)"`
- 检查驱动版本是否满足 CUDA 要求：`nvidia-smi` 顶部显示 CUDA Version
- 推荐使用 Conda 安装 PyTorch 而非 pip

### Jupyter Lab 无法启动
- 确认已激活目标 Conda 环境
- 尝试：`conda install jupyterlab` 而非 pip 安装

## 阶段 6：嵌入式开发环境

### Keil MDK 安装后 uVision 无法打开
- 以管理员身份运行安装程序
- 检查 Windows 安全软件是否拦截

### STM32CubeMX 生成代码后 Keil 编译报错
- 确认 Project Manager 中 Toolchain / IDE 选择了 MDK-ARM V5
- 确认已安装对应 MCU 的器件包（在 CubeMX 的 Help → Manage Embedded Software Packages 中安装）

### 烧录时提示 ST-Link 未找到
- 确认 ST-Link 驱动程序已安装
- 检查开发板与电脑的 USB 连接
- 在 Keil 中确认 Debug → Settings → Debug 选择 ST-Link Debugger
