# 常见问题与踩坑记录

## 阶段 0：网络与账号

### Clash Verge 安装后无法启动
- 以管理员身份运行
- 检查系统防护软件是否拦截

### GitHub 无法访问
- 确认 Clash 已开启系统代理
- 在 Clash Verge 中开启 TUN 模式

## 阶段 1：Git

### `git push` 提示 Permission denied
- 未配置 SSH 密钥，执行 `ssh-keygen` 并添加到 GitHub SSH Keys
- 或切换为 HTTPS 方式：`git remote set-url origin https://...`

### `git commit` 提示 `please tell me who you are`
- 执行以下命令：
  ```
  git config --global user.name "你的名字"
  git config --global user.email "你的邮箱"
  ```

## 阶段 2：Python 与开发环境

### `conda` 命令找不到
- 确认已安装 Miniconda
- 重新打开终端，或执行 conda 的 `condabin/conda.bat`

### pip 安装速度慢
- 确认已配置清华源：`pip config set global.index-url https://pypi.tuna.tsinghua.edu.cn/simple`

### VSCode 找不到 Conda 解释器
- 按 `Ctrl+Shift+P` 打开命令面板
- 选择 `Python: Select Interpreter`，选择对应的 Conda 环境

### ruff 提示大量错误
- 对旧项目先运行 `ruff check --fix` 自动修复
- 新项目定期运行 `ruff check` 和 `ruff format`

### pytest 找不到测试
- 确保测试文件名以 `test_` 开头
- 确保测试函数名以 `test_` 开头
- 在项目根目录运行 `pytest`

## 阶段 3：C 语言、CMake 与 OpenCV

### MSYS2 终端选错
- 安装 UCRT64 套件后，打开 UCRT64 终端，不是 MSYS2 或 MINGW32

### `gcc` 安装后提示找不到
- 执行 `pacman -S mingw-w64-ucrt-x86_64-gcc`
- 确认当前使用的是 UCRT64 终端

### OpenCV cmake 配置失败
- 安装依赖：`pacman -S mingw-w64-ucrt-x86_64-cmake mingw-w64-ucrt-x86_64-make`
- cmake 命令中添加 `-G "MinGW Makefiles"`

## 阶段 4：Linux 虚拟机与容器

### 虚拟机无法联网
- VirtualBox 网络模式设为桥接网络或 NAT
- NAT 模式下宿主机无法直接 SSH 到 VM，需设置端口转发

### SSH 连接被拒绝
- VM 内执行 `sudo apt install openssh-server`
- 确认 SSH 服务已启动：`sudo systemctl status ssh`

### Docker 需要 sudo
- 将当前用户加入 docker 组：`sudo usermod -aG docker $USER`
- 重新登录生效

### Claude Code 安装失败
- 确认 Node.js 版本不低于 18：`node --version`
- 全局安装：`npm install -g @anthropic-ai/claude-code`

### tmux 会话断开后程序停止运行
- 使用 `tmux new -s <会话名>` 启动会话，在其中运行程序
- 分离时使用 `Ctrl+B d`，不要直接关闭终端
- 重连使用 `tmux attach -t <会话名>`

### scp 连接超时
- 确认 SSH 服务已启动：`sudo systemctl status ssh`
- 检查防火墙：`sudo ufw status`
- 确认 VM IP 地址正确：`ip a`

### Shell 脚本无法运行
- 确认第一行是 `#!/bin/bash`
- 添加执行权限：`chmod +x script.sh`
- 使用 `./script.sh` 运行

## 阶段 5：AI / 深度学习环境

### `nvidia-smi` 命令找不到
- 确认已安装 NVIDIA 显卡驱动
- 检查驱动是否与当前 GPU 型号兼容

### PyTorch 安装后 CUDA 不可用
- 确认 CUDA 版本与 PyTorch 版本匹配：`python -c "import torch; print(torch.version.cuda)"`
- 检查驱动版本是否满足 CUDA 要求：`nvidia-smi` 顶部显示 CUDA Version

### Jupyter Lab 无法启动
- 确认已激活目标 Conda 环境
- 使用 `conda install jupyterlab` 安装

## 阶段 6：嵌入式开发环境

### Keil MDK 安装后 uVision 无法打开
- 以管理员身份运行安装程序
- 检查 Windows 安全软件是否拦截

### STM32CubeMX 生成代码后 Keil 编译报错
- 确认 Project Manager 中 Toolchain 选择了 MDK-ARM V5
- 在 CubeMX Help 菜单中安装对应 MCU 的器件包

### 烧录时提示 ST-Link 未找到
- 确认 ST-Link 驱动程序已安装
- 检查开发板与电脑的 USB 连接
- 在 Keil 的 Debug Settings 中选择 ST-Link Debugger

## 阶段 7：学术写作与通用工具

### LaTeX 编译报错 `! Undefined control sequence`
- 检查是否使用了需要 `\usepackage{}` 的宏包
- 常见缺失：`\usepackage{graphicx}`、`\usepackage{amsmath}`

### LaTeX 参考文献不显示
- 按顺序编译：`xelatex → bibtex → xelatex → xelatex`
- 或使用 `latexmk -xelatex paper.tex`

### VSCode 中 Markdown 预览不渲染数学公式
- 在 `settings.json` 中启用 `"markdown.math.enabled": true`

### TeX Live 安装时间过长
- 网络不好时使用清华镜像加速：
  ```
  install-tl-windows.bat --repository https://mirrors.tuna.tsinghua.edu.cn/CTAN/systems/texlive/tlnet/
  ```
